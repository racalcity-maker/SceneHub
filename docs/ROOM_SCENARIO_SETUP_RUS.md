# Настройка Room Scenarios и Game Modes

Room Scenario - главный сценарный движок квеста. Он использует команды и события
Quest Devices, ожидает время, события, оператора, флаги и может завершить игру.

Game Mode выбирает сценарий и длительность игры.

## Общий порядок настройки

1. Создать комнаты в `Rooms`.
2. Добавить Quest Devices в `Device Setup`.
3. Создать Room Scenario в `Scenarios`.
4. Создать Game Mode в `Game Modes`.
5. Выбрать Game Mode в комнате.
6. Запустить игру из Room Control.

## Сценарий

Сценарий принадлежит комнате-владельцу, но шаги могут обращаться к устройствам
из любой комнаты.

Основные поля:

- `name` - имя сценария;
- `room` - комната, из которой этот сценарий запускается;
- `branches` - ветки сценария;
- `steps` - шаги внутри ветки.

Обычный сценарий может быть одной веткой. Для нелинейного квеста можно добавить
до 8 веток.

## Ветки

Ветка линейная внутри себя, но несколько веток могут работать в одной игре.

Поля ветки:

- `name` - имя ветки;
- `enabled` - запускается ли ветка;
- `required_for_completion` - обязательна ли ветка для завершения;
- `steps` - шаги ветки.

Пример:

- `Main` - основной путь;
- `Room 1 side puzzle` - побочная загадка;
- `Basement optional` - необязательная ветка подвала.

Ветки могут синхронизироваться флагами:

1. Ветка A решает загадку и делает `SET_FLAG room1_done=true`.
2. Ветка B решает другую загадку и делает `SET_FLAG altar_done=true`.
3. Ветка C стоит на `WAIT_FLAGS room1_done=true, altar_done=true`.

Так линейные ветки дают нелинейный квест.

## Типы шагов

### DEVICE_COMMAND

Отправить одну команду устройству.

Примеры:

- открыть дверь;
- включить телевизор;
- проиграть аудио;
- сбросить локальную головоломку.

### DEVICE_COMMAND_GROUP

Отправить несколько команд подряд.

Пример:

- открыть два ящика;
- включить телевизор;
- запустить подсветку.

Если одна команда падает, сценарий останавливается на ошибке.

### WAIT_DEVICE_EVENT

Ждать одно событие от одного устройства.

Пример:

- UID-валидатор прислал `completed`;
- реле прислало `drawer_1_opened`;
- алтарь прислал `altar_done`.

### WAIT_ANY_DEVICE_EVENT

Ждать любое событие из списка.

Пример:

- продолжить, если сработал датчик двери или операторский bypass-девайс.

### WAIT_ALL_DEVICE_EVENTS

Ждать все события из списка, в любом порядке.

Пример:

- алтарь готов;
- круги заняты;
- книга положена.

### WAIT_TIME

Пауза в секундах.

Пример:

- подождать 5 секунд после открытия ящика;
- дать аудио начать играть перед следующим действием.

### OPERATOR_APPROVAL

Остановиться и ждать ручного подтверждения оператора.

Используется, когда автоматика не должна решать сама:

- игроки реально прошли этап;
- актер готов;
- можно открыть следующую зону.

### SHOW_OPERATOR_MESSAGE

Показать оператору сообщение и продолжить.

Пример:

- `Prepare basement actor`;
- `Watch camera 3`;
- `Give players hint if stuck`.

### SET_FLAG

Записать флаг внутри текущего запуска игры.

Пример:

- `book_found=true`;
- `altar_done=true`;
- `room2_unlocked=true`.

Флаги сбрасываются при новом старте сценария.

### WAIT_FLAGS

Ждать, пока флаги получат нужные значения.

Пример:

- ждать `book_found=true` и `altar_done=true`, чтобы открыть люк.

### END_GAME

Завершить игру: остановить таймер и перевести сессию в завершенное состояние.

Аудио не останавливается автоматически. Если нужен стоп музыки, добавьте перед
`END_GAME` отдельный шаг `DEVICE_COMMAND -> System Audio -> Stop audio`.

## Game Mode

Game Mode - это режим запуска игры.

Поля:

- `name` - имя режима;
- `room` - комната;
- `scenario` - какой сценарий запускать;
- `duration` - длительность игры;
- `enabled` - доступен ли режим.

Примеры:

- `Easy 60 min`;
- `Hard 45 min`;
- `Debug short run`.

## Проверка перед игрой

Перед стартом сценарий валидируется:

- есть id/name/room;
- все шаги известного типа;
- команды ссылаются на существующие устройства и команды;
- события ссылаются на существующие устройства и события;
- время больше нуля;
- обязательные поля заполнены.

Если есть ошибки, сценарий не должен стартовать.

## Практический пример

Сценарий на 4 комнаты:

1. Ветка `Room 1 main`:
   - ждать UID order success;
   - открыть дверь во вторую комнату;
   - поставить `room1_done=true`.
2. Ветка `Room 2`:
   - ждать `room1_done=true`;
   - открыть два ящика и включить TV;
   - поставить `room2_done=true`.
3. Ветка `Room 3 altar`:
   - ждать события алтаря;
   - ждать датчики кругов;
   - поставить `altar_done=true`.
4. Ветка `Finish`:
   - ждать `room2_done=true` и `altar_done=true`;
   - открыть финальный проход;
   - `END_GAME`.

Так сценарий остается понятным, но квест может идти нелинейно.
