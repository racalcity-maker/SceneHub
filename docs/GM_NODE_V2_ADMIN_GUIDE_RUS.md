# SceneHub GM: админка Node v2

Этот файл описывает практическую работу с Node v2 из GM Panel.

## Что здесь есть

Для одной физической ноды в GM сейчас существуют две разные плоскости работы:

- контракт устройства в SceneHub GM;
- локальный standalone bundle, который хранится на самой ноде.

Это разные вещи и их нельзя путать.

## Где что сохраняется

### `Save device`

Сохраняет в SceneHub GM:

- имя устройства;
- привязку к physical client;
- импортированные resources;
- импортированные commands;
- импортированные events;
- admin templates.

Это не загружает standalone bundle в ноду.

### `Apply bundle`

Сохраняет standalone bundle на ноде.

Это не заменяет контракт устройства в GM.

### `Load stored bundle`

Читает bundle, который реально лежит на ноде сейчас.

Использовать это нужно, когда надо понять:

- что реально записано в ноду;
- совпадает ли браузерный текст с тем, что уже применено;
- не потерялся ли bundle после перезагрузки или смены вкладки.

## Рекомендуемый рабочий порядок

### Импорт возможностей ноды

1. Открыть редактирование quest device.
2. Нажать `Get config`.
3. Проверить counts по resources / commands / events / admin.
4. Нажать `Import`.
5. Нажать `Save device`.
6. Переоткрыть модалку и убедиться, что counts сохранились.

### Работа со standalone bundle

1. Открыть блок `Standalone Rule Engine`.
2. Вставить или загрузить JSON bundle.
3. Нажать `Validate bundle`.
4. Если ошибок нет, нажать `Apply bundle`.
5. При необходимости перезагрузить ноду.
6. Нажать `Load stored bundle` и проверить metadata.

## Как читать состояние

### `ok`

Нода и основные поверхности работают штатно.

### `degraded`

Нода жива, но один из срезов работает с ошибкой. Чаще всего это драйвер
ридера, а не вся нода целиком.

### `offline`

Нет актуальной связи с нодой или конкретным драйвером/подсистемой.

### `fault`

Использовать как более тяжелый сигнал, когда узел реально не способен
обслуживать нормальную работу.

## Частые ошибки оператора

- считать, что `Save device` записывает bundle в ноду;
- считать, что `Apply bundle` обновляет contract в GM;
- не проверять `Load stored bundle` после применения;
- менять bundle и сразу закрывать модалку, не дождавшись результата;
- импортировать config, но не сохранять устройство.

## Если нода в `degraded`

Сначала смотрим detail-строку:

- если там reader `offline` или `init_failed`, это проблема NFC slice;
- если базовые команды работают, ноду не нужно автоматически считать мертвой;
- если driver intentionally absent, можно жить в `degraded` до подключения или
  отключить драйвер в конфиге.

## Что обязательно проверять после изменений

- counts commands / events после повторного открытия модалки;
- наличие нужных admin actions;
- правильность exported scenario commands;
- реальный stored bundle на ноде;
- статус ноды после reboot.
