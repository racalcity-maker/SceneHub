# Node Provisioning Quickstart

Это короткая инструкция для первого ввода ноды в работу.

Полный документ: `PROVISIONING_AND_CONFIG.md`.

## Базовый сценарий

1. Подать питание на ноду.
2. Узнать IP ноды из логов или роутера.
3. Открыть provisioning UI в браузере.
4. Проверить:
   - `Node ID`
   - `Node name`
   - `Operation mode`
   - Wi-Fi
   - host/IP SceneHub
   - MQTT port
5. Сохранить базовую конфигурацию.
6. Перезагрузить ноду.
7. Проверить, что нода появилась в GM.

## Выбор режима

### `scenehub`

Основной managed-режим.

- логика приходит из SceneHub;
- локальный standalone bundle не должен самовольно рулить сценарием.

### `standalone`

Локальная логика живет на ноде.

- bundle валидируется и хранится на ноде;
- MQTT может оставаться включенным, если это разрешено конфигом.

## Что настраивать в первую очередь

- стабильный `node_id`;
- корректный Wi-Fi SSID/password;
- правильный IP/host брокера или SceneHub;
- operation mode под реальный сценарий;
- только те драйверы, которые реально нужны.

## Если включаем PN532

Минимально проверить:

- `enabled=true`;
- I2C SDA/SCL;
- адрес;
- reader id.

Если ридер еще физически не подключен, нода может уйти в `degraded`, но не
должна ломать остальные функции.

## После первого подключения к GM

1. Открыть quest-device modal.
2. Нажать `Get config`.
3. Импортировать compact config.
4. Сохранить device.
5. Переоткрыть модалку и проверить, что counts сохранились.

## Если используем standalone bundle

1. Открыть standalone block.
2. Вставить JSON bundle.
3. `Validate bundle`.
4. `Apply bundle`.
5. При необходимости `Reboot node`.
6. `Load stored bundle` для проверки.
